## 1. 已完成的实验

### 1.1 长文摘要生成基线

- **模型**：`facebook/bart-large-cnn`，采用分块摘要策略（1024 token/块，100 token 重叠，合并后二次摘要）
- **数据**：`ccdv/govreport-summarization` train split，20 个样本
- **运行环境**：CPU only
- **结果**：

| ROUGE-1 | ROUGE-2 | ROUGE-L |
|---------|---------|---------|
| 0.1336 | 0.0453 | 0.0925 |

- **分析**：ROUGE 分数偏低，部分原因在于模型输出的摘要极短（平均 43 词，2.8 句），而参考摘要平均约 200 词。BART 的分块策略导致了信息压缩过度。

### 1.2 句子级事实一致性检测

- **模型**：`facebook/bart-large-mnli`，对每个生成句执行 NLI（前提=原文，假设=摘要句）
- **结果**（55 句，20 样本）：

| 标签 | 数量 | 占比 |
|------|------|------|
| Entailment（一致） | 32 | 58.2% |
| Neutral（无法判断） | 15 | 27.3% |
| Contradiction（矛盾） | 8 | 14.5% |
| **不一致合计** | **23** | **41.8%** |

- **分析**：超过四成的摘要句存在事实一致性问题，其中 contradiction 占比较高（14.5%），意味着这些句子直接与原文事实矛盾。这一结果与已有文献中 BART 类模型约 30% 事实错误率的结论基本一致。

### 1.3 自动纠错

- **方法**：对每个不一致句子，使用词重叠 + 上下文窗口从原文中提取 top-3 证据片段，将证据输入 `facebook/bart-large-cnn` 进行证据引导的抽象式改写
- **结果**：

| | 修复数 | 修复率 |
|---|--------|--------|
| Contradiction → Entailment | 6/8 | 75.0% |
| Neutral → Entailment | 4/15 | 26.7% |
| **总计** | **10/23** | **43.5%** |

- **整体一致性提升**：Entailment 从 58.2% → **76.4%**（+18.2 百分点），不一致率从 41.8% 降至 23.6%

## 2. 初步分析

### 2.1 长度与事实错误率的关系

| 摘要长度 | 不一致率 | 样本数 |
|----------|----------|--------|
| 0-40 词 | 53.8% | 6 |
| 40-60 词 | 38.1% | 14 |

| 原文长度 | 不一致率 | 样本数 |
|----------|----------|--------|
| 0-5,000 词 | 47.8% | 9 |
| 5,000-10,000 词 | 40.7% | 9 |
| 10,000+ 词 | 20.0% | 2 |

**初步结论**：更短的摘要反而错误率更高，可能与信息密度过大、BART 在压缩过多信息时更容易产生幻觉有关。原文越长，BART 的摘要反而更保守、错误率更低——可能是长文包含更多重复信息，BART 更容易找到共识性的关键句。

### 2.2 成功与失败案例分析

**成功案例**（6 例 contradiction 被修复）：
- 典型模式：原句包含具体数字/声明与原文矛盾 → BART 从证据中生成正确的句子
- 例：`[5.1]` "EPA has not adopted an MCL for perchlorate" → 证据显示 EPA 确实未设定 MCL → 改写为正确的上下文句子

**失败案例**（13 例仍不一致）：
- **证据定位错误**（最主要的失败原因）：词重叠方法将不相关段落当作证据
- **neutral 修复难度高**：neutral 本身意味着原文没有足够信息支持或反驳该句，BART 无法凭空创造关联
- **生成冗余**：BART 有时生成过长的多句输出，引入新的不一致

## 3. 后续计划

### 3.1 纠错方法改进（第3-4周）

1. **证据检索升级**：用 TF-IDF 或句子嵌入替换词重叠，提高证据定位准确率
2. **Qwen2.5-1.5B-Instruct 指令纠错**：当前使用 BART 做证据→摘要的间接改写，计划改用 Qwen 做直接的指令纠错，预计修复率可达 60%+
3. **迭代纠错**：对仍未一致的句子进行第二轮证据检索和改写
4. **NLI 重排序**：生成多个候选修正句，选 NLI entailment 分数最高的

### 3.2 分析与报告（第4周）

- 扩展样本量至 50-100（取决于 CPU 时间）
- 整理 10+ 成功/失败案例并做错误分析
- 撰写 ACL 格式终期论文（8-9 页）
- 准备答辩 PPT
